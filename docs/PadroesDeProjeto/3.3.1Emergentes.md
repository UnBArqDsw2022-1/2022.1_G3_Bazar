# Padrões Emergentes - Injeção de Dependência

## 1. Introdução
<p align="justify"> 
Injeção de Dependência é um é um
padrão de projeto usado para evitar o alto
nível de acoplamento de código dentro de
uma aplicação.
Sistemas com baixo acoplamento de
código são melhores pelos seguintes
motivos: Aumento na facilidade de
manutenção/implementação de novas
funcionalidades e também habilita a
utilização de mocks para realizar unit
testes.
O padrão de injeção de dependência visa
remover dependências desnecessárias
entre as classes ou torná-las mais suaves e
ter um design de software que seja fácil de
manter e evoluir.
</p>

## 2. Tipos de Injeção de Dependência
	* injeção por construtor : A injeção de dependência é implementada na definição dos construtores das classes, recebendo por construtor uma instância de classe que irá utilizar.
	* injeção por propriedade (No java getters e setters): implementamos a injeção de dependência na definição dos Gets e Sets das classes.
	* injeção por interface: Aqui a classe deve receber a abstração da implementação que usará por meio de uma interface.
